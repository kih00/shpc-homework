[개요]
LLM의 forward 성능 올리기
- 모델: LFM2-8B-A1B
    - 필요 메모리: 8B X FP32 -> 최소 32 GB
- GPU 사양: TITAN RTX (VRAM: 24 GB) X 4
- 사용 가능 노드 개수: 4개 (노드 1개당 GPU 4개)

[코드 작성 시 절대적으로 지켜야 할 규칙]
- 이유 없이 스켈레톤 코드 훼손 절대 금지
    - 스켈레톤 코드에 기본적으로 선언된 변수들은 무조건 활용하기
    - 스켈레톤 코드의 의미와 의도를 이해하고 코드를 짜기. 이해 못하겠으면 그냥 짜지 말기
- stdout 출력 절대 수정 불가

[Allowed Modifications]
- Changing memory layout
- Reordering loop structures
- Adding padding data or operations,
- Applying operator fusion

[Disallowed Modifications]
- Performing model inference outside the generate function
- Replacing the model with a different one or using a different algorithm that produces the same output.
    - 예: cuBLAS, cuDNN 등 외부 라이브러리 사용 불가
- Skipping model loading based on known expert activations is not allowed, as it assumes unrealistic deployment
conditions

[Basic block]
- 수정 가능한 파일:
    - attn.cu
    - conv.cu
    - moe.cu
- 하나의 GPU에서 돌아감
- 채점 기준:
    - attn: 5000 token/s 이상 만점
    - conv: 10000 token/s 이상 만점
    - moe: 250 token/s 이상 만점

[Full model]
- 수정 가능한 파일:
    - tensor.h, tensor.cu
    - layer.h, layer.cu
    - model.h, model.cu
    - model_loader.h, model_loader.cpp
    - main.cpp
    - run.sh
- 채점 기준:
    - 결과의 로그 스케일로 상대평가

[FP16]
- 기본적으로 FP16을 사용하는 것은 불가능
- 추가 과제로서, FP16 버전의 full model을 구현할 수 있음
- FP16 버전을 구현하여 기존 버전(FP32)의 2배를 넘으면 추가 점수 10%
